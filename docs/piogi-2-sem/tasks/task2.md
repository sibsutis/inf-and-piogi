# Лабораторная работа 2

## Краткое описание

Создание базового каркаса приложения с первым GET-запросом к реальному API.

---

## Обязательные требования

### 1. Layout приложения
- [ ] Окно разделено на две части: список слева, детали справа
- [ ] Используется `Grid` с `ColumnDefinitions`

### 2. Модель данных
- [ ] Создан класс `MovieDto` с полями: `Id`, `Title`, `Description`, `PosterUrl`
- [ ] Переопределён метод `ToString()` для отображения названия

### 3. HTTP-клиент
- [ ] Установлен NuGet-пакет для работы с JSON
- [ ] Создан сервис `ApiService` с методом `GetMoviesAsync()`
- [ ] Используется `HttpClient` для запросов

### 4. Загрузка данных
- [ ] Данные загружаются при старте приложения
- [ ] Метод загрузки асинхронный (`async/await`)
- [ ] Обработка ошибок сети (try-catch)

### 5. Отображение
- [ ] Список фильмов отображается в `ListBox`
- [ ] Данные приходят с реального API (не хардкод)

## Критерии оценивания

| Ранг | Требования |
|------|------------|
| **SILVER** | Приложение запускается, данные загружаются с API, отображаются в списке |
| **GOLD** | Код разбит на классы (ApiService, Models), нет хардкода URL, обработка ошибок |
| **PLATINUM** | Conventional Commits, код чистый и читаемый, есть комментарии |

---

## Технические требования

### API Endpoint
```
GET /api/movies
```

### Ожидаемый JSON
```json
[
  {
    "id": 1,
    "title": "Мстители",
    "description": "Супергерои спасают мир",
    "posterUrl": "https://...",
    "duration": 143
  }
]
```

---

## Критерий сдачи

1. Приложение запускается без ошибок
2. При запуске загружаются данные с API
3. В `ListBox` отображаются названия фильмов
4. Pull Request создан и готов к ревью
