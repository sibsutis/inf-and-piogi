## Часть 2

### 6. Работа с ветками (Git Flow)
- Создана отдельная ветка `feature/lab-1-basic-elements`.
- Все изменения по верстке находятся в этой ветке.
- Ветка отправлена (push) на GitHub.

### 7. Элементы WPF (Полигон)
- В `MainWindow.xaml` реализовано демонстрационное окно ("песочница").
- Использован контейнер компоновки (`StackPanel`, `WrapPanel` или `Grid`).
- На форму добавлены элементы как минимум из 4 категорий:
    - **Buttons**: Кнопка, CheckBox или RadioButton.
    - **Text**: TextBlock и TextBox (или PasswordBox).
    - **Lists**: ComboBox или ListBox (с парой тестовых элементов).
    - **Date**: DatePicker или Calendar.
- Для элементов настроены базовые свойства: отступы (`Margin`), выравнивание (`Alignment`) или цвета.

### 8. Культура работы с Git
- История коммитов в ветке `feature/...` не является пустой.
- Наличие **атомарных коммитов**: добавление каждой группы элементов зафиксировано отдельным коммитом с понятным описанием (например, `Feat: added text inputs`).

## Технические ограничения

!!! danger "Автоматическая пересдача"
    1. Наличие в репозитории папок `/bin`, `/obj` или `.vs` = бан.
    2. Работа велась только в ветке `main` (отсутствие ветки `feature/...`) = бан.
    3. Весь код залит одним коммитом "Final" = пересдача.
