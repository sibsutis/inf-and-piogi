 ## Введение
Одним из ключевых аспектов в современном программировании является **многопоточность**. Ключевым понятием при работе с многоопоточностью является **поток**. 

Поток предствляет некоторую часть кода программы. 

При помощи многопоточности мы можем выделить в приложении несколько потоков, которые будут выполнять различные задачи одновременно. 

>Если у нас, допустим, графическое приложение, которое посылает запрос к какому-нибудь серверу или считывает и обрабатывает большой файл, то без многопоточности у нас бы блокировался графический интерфейс на время выполнения задачи, потому что графический интерфейс "выполняется" в основном потоке. 
>
>А благодаря потокам мы можем выделить отправку запроса или любую другую задачу, которая может долго обрабатываться, в отдельный поток, работающий параллельно. Поэтому, к примеру, клиент-серверные приложения (и не только они) практически не мыслимы без многопоточности.

## Про поток в коде
### **1. Класс `Thread`**

- Это базовый способ создания нового потока.
- Требует ручного управления: запуск, завершение, исключения, синхронизация.
- Используется, когда нужно полный контроль над потоком (например, установка приоритета, фонового режима и т.д.).

```csharp

Thread thread = new Thread(() => {
    Console.WriteLine("Работа в отдельном потоке");
});
thread.Start();


```

### **2. Класс `Task`**

- Это более высокоуровневая абстракция над потоками.
- Работает на основе пула потоков (ThreadPool).
- Упрощает работу с асинхронностью: легко использовать `async`/`await`, можно цеплять задачи (`ContinueWith`), обрабатывать исключения и отмену (`CancellationToken`).
- Используется для параллельных операций и асинхронного программирования.

```csharp

Task.Run(() => {
    Console.WriteLine("Работа в задаче Task");
});

```

> Класс `Task` можно сравнить с `Coroutines` в Unity.
