### Задание 1: Калькулятор с графическим интерфейсом

**Цель:** Создание функционального калькулятора с интуитивно понятным графическим интерфейсом.

**Требования:**

1. **Размещение элементов интерфейса:**

   *   Использование контейнеров `Grid` или `StackPanel` для логичной структуры и гибкости расположения элементов.
   *   Разработка отдельных обработчиков событий для каждой кнопки, обеспечивающих реакцию на действия пользователя.

   Интерфейс должен включать:

   *   **Поле ввода/вывода:** Отображение текущего ввода и результата вычислений.
   *   **Цифровые кнопки (0-9):**  Для ввода чисел.
   *   **Кнопки операций (+, -, *, /):** Для выбора операции.
   *   **Кнопка "=":**  Для выполнения расчета и отображения результата.
   *   **Кнопка "C" (Clear):** Для сброса текущего ввода и результата.

2. **Функционал:**

   *   Обработчики событий:
      *   **Кнопки с числами:** Добавление нажатой цифры к текущему вводу.
      *   **Кнопки с операциями:** Сохранение выбранной операции.
      *   **Кнопка "=":** Выполнение вычислений и отображение результата.
      *   **Кнопка "C":** Очистка поля ввода и сброс результата.
   *   **Последовательность операций:**
      1.  Ввод первого числа.
      2.  Выбор операции.
      3.  Ввод второго числа.
      4.  Нажатие "="" для отображения результата.

**Дополнительно:**

*   **Обработка ошибок:** Предусмотреть обработку исключительных ситуаций, например, деление на ноль, ввод некорректных символов.  Отображать понятные сообщения об ошибках пользователю.  (Например, "Ошибка: Деление на ноль").

### Задание 2: Календарь с выбором периода

**Цель:** Создание календаря, позволяющего пользователю выбирать и просматривать дни выбранного месяца и года.

**Требования:**

1. **Разметка интерфейса:**

   *   Использование контейнера `Grid` для организации сетки дней и `StackPanel` для размещения элементов управления.
   *   Интерфейс должен включать:
      *   **Выбор года и месяца:**  Использование раскрывающихся списков (`ComboBox`) или других навигационных элементов для выбора года и месяца.
      *   **Табличная структура:**  Отображение дней месяца в виде таблицы.

2. **Функционал:**

   *   **Динамическое отображение:**  При выборе года и месяца:
      *   Отображение дней выбранного месяца.
      *   Заполнение первой недели днями предыдущего месяца, если месяц начинается не с понедельника.
      *   Выделение выходных дней (суббота и воскресенье) с помощью визуального акцента (например, красный цвет).

   *   **Обработчики событий:**
      *   **Изменение выбора:** При изменении выбранного года или месяца, автоматическое обновление отображаемого календаря.

*   **Визуальное оформление:**
    *   Стиль для выходных дней (например, изменение цвета фона).
    *   Стиль для дней предыдущего/следующего месяца (например, отображение серым цветом).
*   **Корректное отображение:** Обеспечение корректного отображения для високосных годов.
*   **Навигация:** Предоставить кнопки для переключения между месяцами и годами.

### Задание 3: Погодный виджет

**Цель:** Разработка погодного виджета с возможностью изменения данных через отдельное окно.

**Требования:**

1. **Разметка интерфейса:**

   *   Использование контейнеров для последовательной структуры организации элементов управления.
   *   Интерфейс должен включать:
      *   **Заголовок:** Название города.
      *   **Время и температура:** Отображение текущего времени и температуры.
      *   **Температура и иконка:** Одновременное отображение температуры и соответствующей иконки погоды.
      *   **Блоки информации:**  Отображение данных о ветре, влажности и атмосферном давлении.

2. **Функционал:**

   *   **Отдельное окно конфигурации:** Создание отдельного окна для изменения значений погодных параметров.
   *   **Обновление виджета:**  После внесения изменений в отдельном окне, автоматическое обновление данных на основном виджете.
